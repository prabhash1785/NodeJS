Doctype and HTML

Document type declaration

<!doctype html>
  • Very first line in an HTML document.
  • Tell browsers what mode to render.
  • Also helps when validating your code.

HTML element

<html lang="en">
  • The root of an HTML or XHTML document
  • Should have lang attribute to declare language of document.
  • May have dir attribute to specify the direction of the text.

Head and Title

Head element

<head></head>
  • Provides information about the document: title, metadata, character set, etc.
  • Traditionally, where external resources are referenced.

Title element

<title></title>
  • Container for the title of the page
  • Should be similar to H1 text.
  • What displays in browser tab and shorcut, search results, social media, etc.

Meta

The Meta element has two uses:

  • Contain metadata about the page (with name and content attributes, except charset).
  • Simulate (not override) an HTTP response header (with http-equiv and content attributes).

Examples:

<meta charset="utf-8">
<meta name="description" content="A description about your webpage">
<meta http-equiv="Refresh" content="0;url=http://foobar.com/">

Types:

  • description
  • keywords
  • copyright
  • author
  • robots
  • generator
  • refresh [can use to redirect but 301 is recommended]
  • expires
  • location
  • last-modified
  • allow

Link

Link element

  • Specifies relationship between the current document and external resource
  • Used for external CSS files and application-level icons
  • Attributes include: href, rel, type, media, and sizes (if rel=icon)
  • In HTML5, type attribute not necessary when rel=stylesheet.

Examples:

<link rel="stylesheet" href="/css/foo.css">
<link rel="shortcut icon" href="/images/favicon.ico">
<link rel="apple-touch-icon" sizes="72x72" href="/images/apple-touch-icon.png">

Body and Structure

The body element contains the content of the webpage.

Instead of divs to structure a webpage, use HTML5 structural elements.

Use WAI-ARIA Landmark roles with the HTML5 structural elements.

<body>

<header role="banner">
  <div>My logo</div>
  <p>Slogan here</p>
</header>

<nav role="navigation">
  <ul>
    <li><a href="#">A</a></li>
    <li><a href="#">B</a></li>
    <li><a href="#">C</a></li>
  </ul>
</nav>

<main role="main">
  <h1>Page Title</h1>

  <article>
    <h2>Article Title</h2>
    <p>Ipsum lorem</p>
    <p>Ipsum lorem</p>
  </article>

  <aside role="complementary">
    <h2>Foo</h2>
    <p>Ipsum lorem</p>
  </aside>
</main>

<footer role="contentinfo">
  <p>&copy; copyright 2014</p>
</footer>

</body>

Headings and Sections

Use headings and HTML5 sections

  • H1-H6
  • Heading content defines the header of a section.
  • Recommend one H1 per page (the page title).
  • Always strive for properly nested headings (document outline). Use tools such as the Web Developer Toolbar to test.
  • Benefits readability; accessibility, SEO.
  • Never use text formatting (such as larger font size or bolding) to give the visual appearance of headings; use the H element and modify with CSS.
  • Never overuse the Section element. Should have enough content to require a heading.
<section aria-labelledby="hdWeather">
  <h2 id="hdWeather">Weather report</h2>
  ...
</section>

Tables

Table element

  • Represents data with more than one dimension, in the form of a table
  • Use for data only (not layout)
  • Elements: table, caption, thead, tbody, tfoot, tr, th, td
  • Exercise: think of a hobby and create a data table with related information
<p id="summary">In the following table, negative and positive characteristics are provided.</p>
<table aria-describedby="summary">
 <caption>State of Characteristics</caption>
 <thead>
  <tr>
   <th scope="col">Characteristic</th>
   <th scope="col">Negative</th>
   <th scope="col">Positive</th>
  </tr>
 </thead>
 <tbody>
  <tr>
   <th scope="row">Mood</th>
   <td>Sad</th>
   <td>Happy</th>
  </tr>
  <tr>
   <th scope="row">Grade</th>
   <td>Failing</th>
   <td>Passing</th>
  </tr>
 <tbody>
</table>

Forms

The Form element

"The form element represents a collection of form-associated elements, some of which can represent editable values that can be submitted to a server for processing."

  • Form attributes: action, method, enctype, accept-charset [id, class]
  • Form elements: form, fieldset, legend, label, input, button, select, textarea, meter
  • Input types: text, checkbox, radio, button, hidden, file, search, color, reset, image, tel, url, email, password, date, time, number, range, color
  • Input attributes: autocomplete, autofocus, checked, maxlength, pattern, size, placeholder, required

Tips

  • Use enctype="multipart/form-data" with input type="file".
  • Use autocomplete="off" for security.
  • Use autofocus rather than JavaScript.
  • Placeholder is not a label.
  • Be cautious when using the newer HTML5 input types.
<form action="foo.url" method="post">
  <label for="firstname">First name: </label>
  <input type="text" id="firstname" />

  <label for="lastname">Last name: </label>
  <input type="text" id="lastname" />

  <button type="submit">Send</button>
</form>

Forms cont.

Accessibility

  • Always provide a Label for an input element.
  • Always associate the Label with input element using 'for' and 'id' attributes.
  • Always provide a submit button.
  • Use ARIA for form hints and form validation.
  • Be sure to check your work with multiple browsers (not just Chrome!) as visual treatment (such as focus oulines) may differ.
  • Be aware that if using HTML5, or even ARIA, that a form could still not be ideally accessible.

About fieldset/legend

  • Use Fieldset/Legend to divide long forms and with radio/checkbox groups.
  • When using Fieldset/Legend with radio/checkbox groups, if the Legend contains long text, ARIA can be used instead.
<form>
  <fieldset>
    <legend>Full name</legend>
    <label for="firstname">First name: </label>
    <input type="text" id="firstname" name="firstname" />

    <label for="middlename">Middle name: </label>
    <input type="text" id="middlename" name="middlename" />

    <label for="lastname">Last name: </label>
    <input type="text" id="lastname" name="lastname" />
  </fieldset>
  ...
  <button type="submit">Send</button>
</form>

Divs and Spans

div element - provides a way of dividing a document into meaningful block-level areas (a division)

span element - a generic mechanism for adding in-line structure to documents

Only use when:

  • No other (more semantic) HTML element is appropriate.
  • For styling purposes when absolutely necessary.
<div id="searchModal" role="dialog" class="modal" aria-label="Search" tabindex="-1">
<h2><span aria-hidden="true" class="icon-mail"></span> Email Addresses</h2>

Images

The IMG element

The attributes:

  • src - address of the resource
  • width
  • height
  • alt - replacement text for use when images are not available.
<img src="smiley.png" width="200" height="100" alt="line drawing of a smiley face" />

Tips:

  • Sprites cause accessibility issues (often missing content, don't appear in Windows high-contrast mode).
  • NEVER use background image when the image is not a sprite.
  • Avoid text in images (bad for readability, maintainability, internationalization).

Anchors and Paras

The A element (the hyperlink)

<a href="profile.html">my profile</a>
  • Always use HREF.
  • Always use a valid value in the HREF.
  • Always use textual link content (or image with alt value).
  • The link text should be meaningful out of context.
    For example, a text link should be unique to the page on which it appears; and it's bad practice to use "Click here" or "More".
  • NEVER recreate an element, for example: 
    <div onClick="fool();"></div>

The P element (the paragraph)

<p>This is a line of text in a paragraph.</p>
  • If it's a paragraph (even if one line), use P.
  • If styling appears like a heading, use P with CSS to changes the visible appearance.
  • NEVER use the BR elements to create the visual effect of paragraphs.

XHTML vs HTML5

Syntax differences

XHTML was much more strict. Recommend using the XHTML-type formatting.

  • Elements and attributes must be in lowercase.
  • Attributes must be within quotes.
  • All elements must be closed.

Changed elements and attributes (partial)

  • The cite element now solely represents the title of a work not a person's name.
  • The hr element now represents a paragraph-level thematic break.
  • The small element now represents side comments such as small print.
  • The name attribute on a; use the id attribute instead.

Removed Elements (partial)

  • frame, frameset, noframes
  • acronym (use abbr)
  • applet (use object)
  • big, center, font (use CSS)

What's new HTML5 (partial)

  • Structural elements: header, footer, main, nav, section, aside, article
  • Other new elements: progress, meter, figure/figcaption, datalist, menu, summary/details
  • New form input types: number, color, tel, url, range, email, date, time
  • New attributes: placeholder, autofocus, async, reversed, data-*
  • Audio and Video
  • Canvas
  • Geolocation
  • History API
  • File API
  • Drag & Drop API
  • Web Sockets
  • WebGL (for 3D graphics)
  • Web storage (sessionStorage, localStorage, Web SQL)
  • Offline (cache manifest)
